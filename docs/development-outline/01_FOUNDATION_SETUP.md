# 01 项目基础搭建与工程规范

## 1. 目标

建立前后端分离项目基础骨架，统一目录结构、代码规范、文档规范和开发约束。

## 2. 状态

- 状态：未开始

## 3. 任务清单

- 初始化 `frontend/`、`backend/`、`shared/` 目录
- 建立前后端分离目录规范
- 建立代码风格规范
- 建立英文中文双语注释规范
- 建立依赖引入登记流程
- 建立纯 Go 与避免 CGO 的检查原则
- 建立基础环境变量约定
- 建立基础 Git 提交流程与分支规范

## 4. 完成标准

- 项目目录结构清晰可用
- 文档与目录规范一致
- 后续开发可直接在此基础上推进
